# OpenGL Sorcerer: 3D-Сцена з персонажем

Цей проєкт є 3D-візуалізацією, розробленою на C++ з використанням OpenGL 3.3+ (Core Profile) та бібліотек GLFW, GLAD, GLM та Assimp. Сцена являє собою динамічне середовище з моделлю персонажа, оточеною магічними орбітальними елементами та інтенсивним інтерактивним освітленням.

## Ключові особливості

* **Динамічне освітлення (9 Point Lights):** Реалізовано 9 джерел світла, які динамічно оновлюються у циклі рендерингу.
    * **Центральна Енергетична Сфера:** Головне джерело світла, яке пульсує та змінює інтенсивність/колір, імітуючи магічну енергію. Його параметри керуються користувачем.
    * **Орбітальні Вогняні Кулі (8 шт.):** Додаткові джерела світла, що обертаються по орбіті.
* **Управління Сценою:** Вся логіка анімації, параметрів об'єктів та освітлення інкапсульована в класах `OrbitSystem` та `LightingSystem`, забезпечуючи чистоту `main.cpp`.
* **Інтерактивна Камера:** Камера вільного огляду (FPS-стиль) з регульованою швидкістю переміщення та чутливістю миші.
* **3D Моделі:** Завантаження та рендеринг складних моделей (Dr. Strange, Daggers, Orbs) за допомогою бібліотеки **Assimp**.
* **Skybox:** Реалізовано космічний скайбокс для створення фону.

## Інструкції зі Збірки

Проєкт використовує MinGW-w64 (UCRT64) та конфігурацію VS Code `tasks.json`.

### Необхідні компоненти

1.  **Компілятор:** MinGW-w64 (збірка UCRT64).
2.  **Залежності:**
    * GLFW (Включена як статична бібліотека `libglfw3.a`).
    * GLM (Header-only).
    * **Assimp** (Бібліотека для завантаження моделей. Включена у лінкері).
    * GLAD (Сгенерований лоадер).
    * `stb_image.h` (Для завантаження текстур).

### Структура проєкту

Файли проєкту організовані в підпапки для кращої навігації:
* `src/Core/`: Основні графічні абстракції (`Camera`, `Model`, `Shader`, `Mesh`, `Plane`, `Skybox`).
* `src/Logic/`: Логіка симуляції (`OrbitSystem`, `LightingSystem`).
* `src/Utils/`: Допоміжні функції та глобальні змінні (`Globals`, `Callbacks`, `InputHandler`).
* `src/Vendor/`: Сторонні реалізації (`glad.c`, `stb_image_impl.c`).
* `resources/`: Містить всі 3D-моделі, текстури та GLSL-шейдери.

### Збірка (Visual Studio Code)

Проєкт налаштовано для збірки через **Task Runner** у VS Code.

1.  Переконайтеся, що всі бібліотеки (`lib/`, `include/`) налаштовані правильно, як зазначено у `tasks.json`.
2.  Відкрийте палітру команд (Ctrl+Shift+P або Cmd+Shift+P).
3.  Виберіть **Tasks: Run Build Task**.
4.  Виберіть завдання **`C/C++: Build Project`**.

Це запустить `g++.exe` та скомпілює всі вихідні файли, створюючи виконуваний файл `bin/main.exe`.

## Управління

Камера керується стандартними клавішами FPS-шутера. Параметри сцени контролюються цифровими клавішами в поєднанні з `+` або `-`.

### Управління Камерою
| Дія | Клавіша |
| :--- | :--- |
| Вперед / Назад | **W / S** |
| Вліво / Вправо | **A / D** |
| Вгору / Вниз | **SPACE / LSHIFT** |
| Обертання | **Миша** |
| Зміна швидкості | **1-6** (Без CTRL) |
| Зміна чутливості миші | **CTRL + 1-6** |
| Збільшення/Зменшення FOV | **Скрол миші** |
| Вихід | **ESC** |

### Управління Об'єктами Сцени (Цифрові клавіші + +/-)
| Режим | Клавіші | Дія | Змінні, що керуються |
| :--- | :--- | :--- | :--- |
| **1: Вогняні Кулі** | **1 + (+/-)** | Зміна радіусу та швидкості орбіти вогняних куль. | `fireballRadius`, `fireballSpeed` |
| **2: Кинджали** | **2 + (+/-)** | Зміна радіусу та швидкості орбіти кинджалів. | `daggerRadius`, `daggerSpeed` |
| **3: Центральна Сфера** | **3 + (+/-)** | Зміна висоти, базового масштабу та інтенсивності світла пульсуючої сфери. | `orbHeight`, `orbBaseScale`, `orbIntensity` |
